Collection Framework in Java

The Collection Framework in Java provides a set of classes and interfaces that help store and manipulate groups of objects efficiently. It is located in the java.util package and provides ready-to-use data structures like lists, sets, queues, and maps.

Why Use Collections?

  • To store, retrieve, and manipulate groups of data easily.
  • To avoid using arrays with fixed sizes.
  • To use built-in methods for sorting, searching, and iteration.

Main Interfaces in the Collection Framework

  • Collection — The root interface for most collection classes.
  • List — Ordered collection that allows duplicate elements.
  • Set — Unordered collection that does not allow duplicates.
  • Queue — Follows FIFO (First In, First Out) order.
  • Map — Stores key-value pairs and does not extend the Collection interface.

Hierarchy of the Collection Framework

Iterable → Collection → List, Set, Queue
Map is a separate interface that works independently.

Commonly Used Classes

  • ArrayList — Dynamic array that allows random access of elements.
  • LinkedList — Implements a doubly linked list; better for frequent insertions/deletions.
  • HashSet — Stores unique elements and allows fast lookup using hash values.
  • TreeSet — Stores unique elements in sorted order.
  • HashMap — Stores data in key-value pairs for quick access.
  • LinkedHashMap — Maintains insertion order for key-value pairs.

Example: Using ArrayList


import java.util.ArrayList;

public class ArrayListExample {
    public static void main(String[] args) {
        ArrayList list = new ArrayList<>();

        list.add("Java");
        list.add("Python");
        list.add("C++");

        System.out.println("Programming Languages:");
        for (String lang : list) {
            System.out.println(lang);
        }
    }
}

Example: Using HashMap


import java.util.HashMap;

public class HashMapExample {
    public static void main(String[] args) {
        HashMap map = new HashMap<>();

        map.put(1, "Apple");
        map.put(2, "Banana");
        map.put(3, "Cherry");

        System.out.println("HashMap Elements:");
        for (Integer key : map.keySet()) {
            System.out.println(key + " => " + map.get(key));
        }
    }
}

Iterating Over Collections

Collections can be traversed using iterators or enhanced for loops. 


import java.util.*;

public class IterateExample {
    public static void main(String[] args) {
        List numbers = Arrays.asList(10, 20, 30);

        System.out.println("Using Iterator:");
        Iterator it = numbers.iterator();
        while (it.hasNext()) {
            System.out.println(it.next());
        }

        System.out.println("\nUsing For-Each Loop:");
        for (int num : numbers) {
            System.out.println(num);
        }
    }
}

The Collection Framework simplifies data management by providing ready-made data structures and algorithms for efficient data storage, retrieval, and manipulation.
