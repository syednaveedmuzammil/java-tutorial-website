Object-Oriented Programming (OOPs) Concepts in Java

Java is based on the Object-Oriented Programming (OOP) approach, which focuses on organizing a program around objects rather than functions and logic. This helps in making the code more reusable, modular, and easier to maintain.

1. Class

A class is a blueprint or template that defines the structure and behavior of objects. It contains variables and methods that describe what an object can hold and do. 


// Example: Defining a class
public class Student {
    String name;
    int age;

    void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
    }
}

2. Object

An object is an instance of a class. Each object has its own state and behavior defined by its class. Objects are created using the new keyword in Java. 


// Example: Creating and using an object
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();   // Object creation
        s1.name = "Umesh";
        s1.age = 22;
        s1.displayInfo();
    }
}

3. Abstraction

Abstraction means hiding the internal details and showing only the essential features of an object. It focuses on what an object does instead of how it does it. Abstraction in Java can be achieved using abstract classes and interfaces. 


// Example: Abstraction
abstract class Vehicle {
    abstract void start();
}
class Car extends Vehicle {
    void start() {
        System.out.println("Car starts with a key");
    }
}

4. Encapsulation

Encapsulation is the process of wrapping data and methods into a single unit. It restricts direct access to variables by declaring them private and provides controlled access through getter and setter methods. 


// Example: Encapsulation
public class Account {
    private double balance;

    public void setBalance(double balance) {
        this.balance = balance;
    }

    public double getBalance() {
        return balance;
    }
}

5. Inheritance

Inheritance allows one class to acquire properties and methods of another class. It promotes code reusability and represents an “is-a” relationship between classes. 


// Example: Inheritance
class Animal {
    void eat() {
        System.out.println("Eating...");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("Barking...");
    }
}

6. Polymorphism

Polymorphism means “many forms”. It allows one interface to be used for multiple purposes. It is achieved in Java through method overloading (compile-time) and method overriding (runtime). 


// Example: Polymorphism
class Shape {
    void draw() {
        System.out.println("Drawing a shape");
    }
}
class Circle extends Shape {
    void draw() {
        System.out.println("Drawing a circle");
    }
}

OOP in Java helps in organizing code better and makes applications more flexible and scalable.
